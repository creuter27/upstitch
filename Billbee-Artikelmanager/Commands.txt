Summary of what the tool does:
For each listing-format row with BOM SKUs, it parses both sides and compares manufacturer, category, variant, size (compound-aware), and color
For compound listing sizes like bs (big+small), it extracts the per-item expected token by index: BOM[0]→b, BOM[1]→s
Proposes a corrected BOM SKU by swapping the wrong slot value using the listing's raw token
Approved fixes update the BOM_SKUs column in the sheet (Billbee is not changed — this stages the corrections for a future upload step)
--dry-run shows all mismatches without prompting
--listing-id PREFIX filters to a specific listing ID

Usage for Trixie:

Step 1 — login once:
.venv/bin/python execution/b2b_cart.py setup --manufacturer TRX --url https://b2b.trixie-baby.com/
    
Step 2 — scrape the catalog once:
.venv/bin/python execution/b2b_cart.py explore --manufacturer TRX
If there is a page that contains all products (Trixie):

.venv/bin/python execution/b2b_cart.py explore --manufacturer TRX \
    --url https://b2b.trixie-baby.com/products-1 --no-crawl

With --no-crawl, only the given URL is scraped (with full load-more scrolling). No links are followed. 
For Trixie this is the /products-1 SHOP page that lists all 949 products.

Reorder process:
Repeat Step 2 — re-scrape the catalog if necessary:
.venv/bin/python execution/b2b_cart.py explore --manufacturer TRX --refresh
Step 3 — order (after mapping is filled):
.venv/bin/python execution/b2b_cart.py order --manufacturer TRX --dry-run
Adding Fresk later:

Skip list (.tmp/TRX_skip.txt):

map command (also runs automatically after a successful explore):

.venv/bin/python execution/b2b_cart.py map --manufacturer TRX

Reads the mapping tab (Kategorie / Größe / Herstellername / Avoid)
Reads the TRX B2B Catalog tab
Reads the ProductList tab
For each ProductList row missing an EAN: finds the matching B2B product using the mapping + variant tokens from products.yaml
Batch-writes matched EANs into the EAN column
